1、使用 AutomationTool 对项目打包， 生成球场pak包
   1）使用方法为： 先在Project Launcher中设置好两个profile（一个用于项目打包，一个用于生成球场的dlc包）。
   2）之后从Project Launcher的log中拷贝出需要的命令行。（第四行前面带着 Parsing command line的, 从 Parsing command line：后的内容为命令行）
   3）之后打开cmd定位到引擎中AutomationTool所在目录（Engine/Binaries/DotNET目录）
   4) 将-targetplatform=Win64 改为 将-targetplatform=Win32 即可生成32位的包
   5）输入 start AutomationTool.exe 后面接刚刚从Project Launcher中复制过来的命令
行

得到一句很长的命令，类似这种
start AutomationTool.exe -ScriptsForProject=E:/UnrealProjects/movementCompClearVer/movementComp.uproject BuildCookRun -project=E:/UnrealProjects/movementCompClearVer/movementComp.uproject -noP4 -clientconfig=Shipping -serverconfig=Shipping -nocompile -nocompileeditor -installed -ue4exe=UE4Editor-Cmd.exe -UseDebugParamForEditorExe -utf8output -platform=Win64 -targetplatform=Win32 -build -cook -map=hole01+hole02+hole03+hole04+hole05+hole06+hole07+hole08+hole09+hole10+hole11+hole12+hole13+hole14+hole15+hole16+hole17+hole18 -unversionedcookedcontent -pak -dlcname=FirstDLC -basedonreleaseversion=2.0 -compressed -stage -package -cmdline=" -Messaging" -addcmdline="-SessionId=B8583E3C4F162FA384578F8195BDF47E -SessionOwner='Administrator' -SessionName='TestDlc'"


   6) DLC包需要用到Release版本的AssetRegistry.bin文件 保存一个原始的AssetRegistry.bin文件用于DLC打包即可
      同时，DLC包还需将素材复制到发布工程里才能完成打包。不可单独打包。

   7）DLC包只需与发布版的pak文件放到一起，即可自动加载

   8) 对于小地图和热度图，由于在编辑器中，不会用到这两张贴图，所以，打包的时候，不会包含在内。 所以，需要一个小技巧。 在地平面下，创建一个方块，为它贴上材质，然后，在这个材质里，引用热度图和小地图; 或者将球场文件夹中的tietu文件夹 加入 Project Settings-》Packaging-》Addtional Asset Directories to Cook中

将generate Patching改成 Build DLC即可生成DLC包

若产生了，Couldn't Update Resource的错误，可以遵循以下步骤 解决
	1、将 Package 选项改为 do not package
	2、Anchieve 勾上， 随便选择一个路径
然而，这个错误不影响DLC打出来的pak包，即使报了这个错误，也可以忽略掉
